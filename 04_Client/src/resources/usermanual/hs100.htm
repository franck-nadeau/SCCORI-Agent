Rules Based (Pull Spike) Agent

This is a "seeing" agent that knows all of the other players stocking levels. It should only be used in Pull simulations.

 

Rules Based (Pull Spike) agent uses the same rules as the Rules Base (Pull Random) agent after the spike, but it does its best to maximize stock before the spike. It does its best to maintain "adequate" stocking levels. This agent tries to minimize shipping costs after the spike and it avoids back logging its supplier - as long as the supplier is attempting to maintain adequate stocks itself.

 

The Rules Based (Pull Spike) Agent uses the following parameters:

  • maximum inventory depends on the position played and pre/post spike
  • before spike, all positions = 100
  • retailer = 100
  • wholesaler = 80
  • distributor = 80
  • manufacturer = 65 (higher previous stocking levels allow lower levels here)
  • restocking trigger level = 40 (ignored before the spike)
  • container size = 60
  • minimum order = 30 (order half a container or more)
